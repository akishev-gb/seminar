# Инструция по начальным командам в Git

## Команды для работы в системе контроля версий
<!--Commands-->
*git init* - **команда инициализации репозитория локально**.

* **Репозиторий** - это папка с файлами и настройками *Git*. Настройки расположены в скрытой папке *.git*.

*git add* - **команда для добавления изменений или добавление файла в отслеживание**.
- При помощи табуляции и введения первых двух символов названия нашего файла, отобразится имя файла целиком. Упрощает ввод команды через терминал.

*git commit -m "some message"* - **команда для добавления коммита(закрепление изменений в репозитории)**.
* **Коммит(Commit)** - фиксация изменений или запись изменений в репозитории. Коммит происходит на локальном компьютере. Иначе говоря: **коммит** - это как ~~save~~ в игре.
- *git commit --amend -m "Правильное сообщение коммита"* - _комманда для исправления последнего коммита_.

*git log* - **команда для вывода журнала изменений**.
 * Иными словами покажет всю историю *коммитов* этого репозитория.<br> В консоли появится информация подобного рода: ![image](log.png)<br>* Непонятный набор символов - это **хэш** коммита. Зная **хэш** коммита можно заглянуть в лог коммита, либо откатиться к данному коммиту.

*git status* - **команда для просмотра локальных изменений**.
-  При помощи этой команды можно проверить, закомитились ли файлы. Если да, будет сообщение такого плана: **"nothing to commit, working tree clear".**

*git diff* - **команда для просмотра различий между изменениями**.
 * Эта команда покажет что именно изменилось в файле: красным - удаленное, зеленым - добавленное.

*git checkout (master)* - **команда для перемещения между коммитами, иными словами откатиться до определенного коммита**.
 * **Мастер(Master)** - главная или основная ветка репозитория.
  * Для перемещения, либо отката до определенного коммита, достаточно в команду *git checkout* ввести первые 4 символа **хэша**.
  
  *git clone* (Клонить) - _комманда для скачивания удаленного репозитория на локальный репозиторий_.
  
  *git pull* (Пул) - _комманда для получения последних изменений с удаленного сервера репозитория_.

  *git push* (Пуш) - _комманда для отправки всех неотправленных коммитов на удаленный сервер репозитория_.

  *pull request* (Пул реквест) - _запрос на слияние __форка__ репозитория с основным репозиторием. Пул реквест может быть принят либо отклонен основном владельцем репозитория_.
  - **Fork (Форк)** - копия репозитория. [GitHub](https://github.com/) создаст копию форкнутого репозитория у вас на аккаунте. После этого в копию этого репозитория можно будет вносить изменения. Внесенные изменения можно предложить добавить в основной репозитория через пул реквест.

*git merge Branch_name* (Мёрдж) - _комманда для слияния изменений из какой-либо ветки репозитория с любой веткой этого же репозитория. Чаще всего - слияение изменений из ветки репозитория с основном веткой репозитория_.

**Branch (Ветка)** - вся история коммитов приводящая в текущую точку.

*git branch* - __комманда для вывода веток__.

*git branch branch_name* - __комманда для создания ветки__.

*git checkout branch_name* - __комманда для перехода на ветку__.

*git checkout -b branch_name* - __комманда для перехода в свеже-созданную ветку__
