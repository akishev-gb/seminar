# Инструкция по работе с GIT.

## 1.  Начальные настройки и установки

(В данном пункте будут представлены базовые функции, которые используются для работы с системой контроля версий)

---

+ Сперва нужно авторизоваться в ```git ```. Для этого необходимо выполнить следующие команды <<*```git config --global user.name «Ваше имя английскими буквами»```*>>
 например: **Nikita** и <<*```git config --global user.email ваша почта@example.com```*>>

---
+ После успешной авторизации нужно провести инициализацию системы, для начала работы. В этом нам поможет команда
<<*```git init```*>>– **инициализация локального репозитория**

---
+ Чтобы проверить статус программы мы можем воспользоваться командой
<<*```git status```*>> – **получить информацию от git о его текущем состоянии**
![git status](git_status.png)

---
+ Далее мы можем создать рабочую папку и в ней создать файлы, в которых будем работать
**Очень важно не забывать сохранять свои действия в файлах!!!**
![work_place](work_place.PNG)
---
+ Теперь нам необходимо, чтобы система определила и запомнила файл, который мы написали в файле. Для этого нам помогут команды
<<*```git add```* – **добавить файл или файлы к следующему коммиту**
и
<<*```git commit -m “message”```*>> – **создание коммита**

---
+ Чтобы посмотреть историю всех сохранений мы можем воспользоваться командой
<<*```git log```*>> – **вывод на экран истории всех коммитов с их хеш-кодами**
![git_log](git_log.PNG)

---
+ Также мы можем переходить между сохранениями (или ветками, о которых будет описано позже) с помощью команды
*```git checkout```* – **переход от одного коммита к другому**
А чтобы вернутся к актульному состоянию, необходимо к нашей команде дописать
<<*```git checkout master```*>> – **вернуться к актуальному состоянию и продолжить работу**

---
+ Для того, чтобы оценить разницу между сохранениями существует команда
<<*```git diff```*>> – **увидеть разницу между текущим файлом и закоммиченным файлом**

## 2. Работа с ветветвлением в GIT

---
* Для создания веток воспользуемся командой
*```<<git branch (название ветки)>>```* - **команда для создания новой ветки**

* Чтобы посмотреть список всех веток введем туже самую команду, но без названия ветки *```<<git branch>>```* - **команда для вывода всех веток на экран**
![git_branch](git_branch.PNG)

---
* Чтобы переключаться между ветками напишем команду *```<<git checkout (название ветки)>>```* - **команда для переключения между ветками**

---
* Чтобы удалить ненужные ветки существует команда *```<<git branch -d (название ветки)>>```* - **команда для удаления ветки**

---
* Также существует возможность слияния веток, для этого есть команда *```<<git merge название ветки)>>```* - **команда для слияния веток**. При объединение веток может случиться конфликт объединения. Это связано с тем, что в этих двух вариантах на тех же строках присутствуют какие-то значения.
![git_merge_second_seminar](git_merge_second_seminar.PNG) 
---
* Посмотреть историю коммитов с визуализацией поможет команда *```<<git log --graph>>```* - **команда для вывода истории коммитов с визуализацией**
![git_log_graph](git_log_graph.PNG)

---
* Теперь мы слили два файла семинара в один файл мастера и возник конфликт. В данном случае сохраняем оба варианта.
![git_merge_both_seminar](git_merge_both_seminar.PNG)

## 3.  Работа с удаленными репозиториями

В системе контроля, помимо работы с локальными данными (которые находятся на нашем устройстве) существует также возможность работы с удаленными данными (которые расположены вне\за пределами нашего устройства)
---
+ Если источник имеет открытый доступ, то мы можем создать клон этих данных на своей локальное устройство. Для этого существует команда *```<<git clone (ссылка)>>```* - **клонировать репозиторий из удаленного источника**. На картинке указан результат клонирования файла в терминале.
![git_clone](git_clone.PNG)

---
+ Чтобы отправить на удаленный репрезиторий из локального устройства мы воспользуеся командой *```<<git push>>```* - **отправка изменений на удаленный репозиторий**
![git_remote_add_origin](git_remote_add_origin.PNG)

---
+ Мы можем работать с данными через удаленный репрезиторий.После того, как мы внесли изменения, то с помощью команды *```<<git pull>>```* - **команда для подгрузки изменений из удаленного репозитория** можно подгрузить актуальную версию на свое локальное устройство.
![git_pull](git_pull.PNG)
---
+ Мы также можем привязать удаленный репозиторий к локальному репозиторию с помощью команды *```<<git remote add origin (ссылка)>>```* - **привязка удаленного репозитория к локальному репозиторию**
