# Инструкция по работе с Git

![GIT лого](GIT.jpg)

## Первый запуск

При первом запуске GIT пользователь **обязательно** должен *"Представиться"*.

Для этого требуется ввести следующие команды:

1. git config --global user.name **_«Ваше имя английскими буквами»_**

Например: Evgeniy

2. git config --global user.email ваша почта@example.com

Данные не проверяются, то есть можно указать любую почту и тем более любое имя. Но **лучше написать реальные данные** или, как минимум что-то осмысленное). *Эти данные будут указываться, когда вы будете делать коммиты, а указание реальных данных упростит командную работу*.

## Основные команды

*git --version* - **Вывод версии системы контроля версий**
> В старых версиях GIT обязательно использование "--", в более новых версиях GIT допустимо использовать *git version*

*git init* – **инициализация локального репозитория**

*git status* – **получить информацию от git о его текущем состоянии**

*git add* – **добавить файл или файлы к следующему коммиту**

*git commit -m “message”* – **создание коммита**.

*git log* – **вывод на экран истории всех коммитов с их хеш-кодами**

*git diff* – **увидеть разницу между текущим файлом и закоммиченным файлом**

*git checkout* – **переход от одного коммита к другому**

*git checkout* master – **вернуться к актуальному состоянию и продолжить работу в основной ветке**

## Команды для работы с ветвлением

*git branch* - **Вывод всех веток на экран**

*git branch new_branch_name* - **Создание новой ветки с именем _new_branch_name_**

*git checkout branch_name* - **Переход на ветку branch_name**

*git checkout -b branch_name* - **Создание и переход на новую ветку branch_name**

*git merge branch_name* - **Объединить ветку с именем _branch_name_ с текущей веткой**

*git branch -d branch_to_delete* - **Удалить ветку с именем _branch_to_delete_**

*git log --graph* - **Вывести список коммитов в виде красивого графа/дерева**

## Команды для совместной работы и обновления проектов

*git clone* - **Скопировать внешний репозиторий на свой ПК. _Не только загружает все изменения, но и пытается слить все ветки на локальном ПК и в удаленном репозитории_**

*git pull* - **Забрать изменения из указанного удалённого репозитория, а затем пытается слить их с текущей веткой**

*git push* - **Отправить свою версию репозитория во внешний репозиторий. _При первом использовании требует авторизации_**

## Дополнительные команды терминала

*clear* - **Очистить экран терминала** 

*Стрелка вверх/стрелка вниз* - **Быстрое переключение между ранее введенными командами**

*Кнопка TAB* - **Позволяет продолжить написание имени файла/папки после трех символов**

## Заголовки Markdown

![markdown лого](markdown.jpg)

Для использования **Заголовка** в языке Markdown, требуется начать строку заголовка со знака решетки и пробела *"# "* 

Для использования **Подзаголовка** в языке Markdown, требуется начать строку подзаголовка с двойного знака решетки и пробела *"## "* 

## Выделение текста Markdown

Для выделения текста курсивом текст обрамляется звездочками "*" или знаком нижнего подчеркивания "_" 

Пример: *Курсив* или _Курсив_

Для изменения шрифта на полужирный текст обрамляется двойными звездочками "**" или двойным знаком нижнего подчеркивания "__" 

Пример: **Полужирный** или __Полужирный__

Можно применять комбиинации форматирования текста с помощью использования разных символов при форматировании

Пример: _**Невероятно!** Это было не очевидно__

## Списки Markdown

Для построения ненумерованных списков используется "* " или знак "+ " в начале каждой строки списка.

Пример:

* Элемент1
* Элемент2
* Элемент3
+ Элемент4
+ Элемент5

Для построения ненумерованных списков используется "<Цифра>. " в начале каждой строки списка.

Пример:

1. Первый элемент списка
2. Второй элемент списка

## Работа с изображениями

Для вставки изображения в текст используется следующая конструкция:

!\[]\()

В [] указывается сообщение для вывода в случае невозможности отображения картинки.

В () указывается пути и имя файла. Если файл размещен в той же папке, достаточно указать только имя файла.

## Работа с таблицами

Для вывода в формате таблиц используются вертикальные линии "|", которые обозначают столбцы.

Пример:

| Таблицы  | Это   | Круто |
| -------- |:-----:| -----:|
| A1       | B1    | $2000 |
| A2       | B2    |   $11 |
| A3       | B3    |    $1 |

Внешние вертикальные линии (|) не обязательны и нужны только, чтобы сам код Markdown выглядел красиво. Тот же код можно записать так:

Markdown | не такой | красивый
--- | --- | ---
*Но выводится* | `так же` | **эффективно**
1 | 2 | 3

## Цитаты

Для выделения цитаты в начале каждой строки цитаты используется символ ">"

Пример:

> С помощью цитат очень удобно в письме обозначать исходный текст.
> Эта строка - часть той же цитаты.

Разрыв цитаты.

> Это очень длинная строка, но она будет правильно процитирована даже при размещении на нескольких строках. Продолжаем писать, чтобы эта строка не вмещалась на одной строке в любом окне. Кстати, в цитаты можно *вставлять* даже **Markdown**.

## Программный код

Программный код обрамляется с помощью символов "`"

Пример:

`Код` в строке обрамляется `обратными апострофами`

*Блоки кода* выделяются либо тремя обратными апострофами ``` либо четырьмя пробелами в каждой строке. Рекомендуется использовать три апострофа -- они проще и только они поддерживают подсветку синтаксиса

```javascript
var s = "Подсветка JavaScript";
alert(s);
```
 
```python
s = "Подсветка Python"
print s
```
 
```

Но мы вставим в него <b>тег</b>.
```
